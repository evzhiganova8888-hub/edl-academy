# Деплой `masterskaya.html` — инструкция

Файл готов и кладётся в существующий репозиторий рядом с `index.html`, `sprint.html`, `diagnostic.html`. Он переиспользует `styles.css` и `main.js` — отдельных файлов не нужно. Все шрифты (Plus Jakarta Sans, Geist, Geist Mono) подгружаются через Google Fonts, как и на остальных страницах.

---

## Что внутри страницы

9 блоков, под TOV v1.0 и Brand Guidelines v1.0:

1. **Hero** — статус-чип «Пилотный поток · июнь 2026», заголовок с lime-акцентом на «услышали», CTA «Обсудим вашу идею лекции»
2. **Откуда идея** — четыре цитаты из кастдевов (Лена, Денис, Николай, Константин) — единственное реальное доказательство спроса до PMF-теста
3. **Для кого** — три карточки в стиле `funnel-card`: эксперт без формата, автор без голоса, будущий лектор
4. **Программа** — пять занятий с большими номерами, по 90 минут
5. **Чего не делаем** — DEEP-секция с тремя зачёркнутыми обещаниями (lime-полоса перечёркивания): «курс публичных выступлений», «как продать себя», «упрощайте контент»
6. **Что после** — путь из четырёх шагов с выделенной финальной карточкой (запуск через Академию)
7. **Условия** — таблица: цена 12 000–15 000 ₽, формат, размер группы, старт, преподаватель, возврат
8. **FAQ** — 5 вопросов через стандартный механизм `main.js`
9. **Final CTA** — DEEP-секция, дублирует Telegram-канал

## Деплой через веб-интерфейс GitHub — 4 шага

Подходит, если работаете без локального git.

1. **Открыть репозиторий сайта** на GitHub (тот, в котором уже лежат `index.html`, `sprint.html`, `styles.css`)
2. **Нажать `Add file → Upload files`** → перетащить `masterskaya.html` в окно
3. **Внизу страницы:** в поле commit message написать `add: Мастерская Авторов · v1` → нажать **Commit changes**
4. Подождать ~30–60 секунд — GitHub Pages пересоберёт сайт. Страница будет доступна по адресу:
   ```
   https://elephantdreams.ru/masterskaya.html
   ```

Никаких настроек в `Settings → Pages` менять не надо — деплой уже включён для этого репо.

## Что обновить на остальном сайте — 5 минут работы

Чтобы новая страница не была сиротой, добавьте на неё ссылки в трёх местах:

### 1. Шапка — `index.html`, `sprint.html`, `diagnostic.html`

Найти блок:
```html
<nav class="header__nav" aria-label="Основная навигация">
  <a href="index.html#approach">Подход</a>
  <a href="sprint.html">Спринт</a>
  <a href="diagnostic.html">Диагностика</a>
  <a href="https://t.me/elephantdreamscommunity" ...>Telegram</a>
</nav>
```

Добавить строку перед Telegram:
```html
<a href="masterskaya.html">Мастерская</a>
```

То же самое в блоке `<nav class="mobile-menu">` ниже — там список `<li>`.

### 2. Подвал — те же три страницы

Найти блок «Услуги» в `<footer>`:
```html
<ul class="footer__list">
  <li><a href="sprint.html">AI Operations Sprint</a></li>
  <li><a href="diagnostic.html">Диагностика</a></li>
  <li><a href="sprint.html#growth">Growth Plan</a></li>
</ul>
```

Заменить на:
```html
<ul class="footer__list">
  <li><a href="sprint.html">AI Operations Sprint</a></li>
  <li><a href="diagnostic.html">Диагностика</a></li>
  <li><a href="masterskaya.html">Мастерская Авторов</a></li>
</ul>
```

### 3. `sitemap.xml`

Добавить запись:
```xml
<url>
  <loc>https://elephantdreams.ru/masterskaya.html</loc>
  <lastmod>2026-04-27</lastmod>
  <changefreq>weekly</changefreq>
  <priority>0.8</priority>
</url>
```

## Что НЕ вошло в эту версию (намеренно)

Это MVP — собран по принципу «достаточно для PMF-теста». В следующих итерациях:

- **EN-версия** (`en/masterskaya.html`) — сейчас ссылка ведёт в 404. Перевод имеет смысл делать после первого потока, когда финальный текст устаканится.
- **OG-image** — сейчас используется общий `og-image.png`. Своя картинка нужна, когда страница начнёт расходиться по соцсетям.
- **Преподаватель — имя и фото** — добавляется в блок «Условия» после подписания контракта (до конца апреля по плану из Excel).
- **Календарь потоков** — сейчас один пилот в июне. Когда план на сентябрь/январь зафиксируется — добавится секция «Расписание потоков».
- **Социальное доказательство после первого потока** — отзывы участников, цифры конверсии в Академию. Появляются в августе.

## Технические детали (если интересно)

- **Файл:** один HTML, ~38 КБ, 820 строк. Без зависимостей кроме `styles.css` и `main.js`, которые уже в репо.
- **Стили:** реюзают всю дизайн-систему (`section--tinted`, `funnel-card`, `eyebrow`, `btn--primary`, `faq-item`). Свои добавления — в `<style>` блоке внутри `<head>`, под префиксом `.ms-*` (около 200 строк, изолированы).
- **JS:** ничего нового — FAQ, модалка, sticky-CTA, mobile-menu обрабатываются существующим `main.js`.
- **Шрифты:** те же три, что на остальном сайте.
- **Доступность:** `aria-*` атрибуты, skip-link, корректная иерархия H1→H2→H3, `<button>` для интерактивных элементов FAQ.
